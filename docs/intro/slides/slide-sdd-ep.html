SDD vs Executing-Plans

维度
Subagent-Driven Development
Executing-Plans
执行
每 task 启动一个全新 subagent，独立执行
当前 session 内逐 task 执行，保持连续上下文
审查
两阶段 review：spec 审查 + 质量审查，双重点检
checkpoint 审查：每个关键节点停下来验证，逐 task 推进
上下文
每个 subagent 独立隔离，互不干扰，无状态泄露
共享 session，前后 task 可传递中间产物和上下文
适用
task 之间互相独立，对严格质量有要求
task 之间强耦合，依赖连续上下文传递

Alloy 根据 task 的耦合度自动选择执行模式：互不依赖的任务走 SDD（并行+隔离+双审查），强耦合链路走 Executing-Plans（连续上下文+checkpoint 验证）
