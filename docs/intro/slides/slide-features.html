6 个核心特性

01 流程闸门，不是建议

跳过 TDD 被脚本阻断，跳过审查 hash 对不上。硬门禁不是"建议"，是自动校验的脚本关卡。

02 OpenSpec + Superpowers

一条命令搞定，不用学 20+ 个技能。需求管理 + 流程闸门在一条流水线里自动编排。

03 随便打哪个命令，都能接上

自动路由 + 断点恢复。会话断了、换了个命令？系统自动识别当前阶段，无缝续接。

04 改需求不乱

还没写就回溯到上一个阶段，已经写了就开新 change。Delta Spec 机制保证变更可追踪。

05 每次做完，下次更聪明

Retrospective 教训跨周期传递。每次合入后自动沉淀经验，下个周期 Agent 自动参照改进。

06 每一步都可追溯

制品 hash 锁定 + 独立 commit，完整审计链。谁改了什么、为什么改、什么时候改，全部可查。
