Step 1 环境感知
→
Step 2 根因诊断
→
Step 3 分支选择 + 修复
[HARD STOP] 禁止在主分支上直接修改代码 — 一行和千行的保护等级一样
SCENE 1
Worktree 内修复
phase=applied 且 worktree 存在

SCENE 2

applied(无 worktree) 或 planned

SDD vs Executing-Plans

两种执行模式，根据 task 耦合度自动选择

维度Subagent-Driven DevelopmentExecuting-Plans
执行每 task 启动一个全新 subagent，独立执行当前 session 内逐 task 执行，保持连续上下文
审查两阶段 review：spec 审查 + 质量审查，双重点检checkpoint 审查：每个关键节点停下来验证，逐 task 推进
上下文每个 subagent 独立隔离，互不干扰，无状态泄露共享 session，前后 task 可传递中间产物和上下文
适用task 之间互相独立，对严格质量有要求task 之间强耦合，依赖连续上下文传递

Alloy 根据 task 的耦合度自动选择执行模式：互不依赖的任务走 SDD（并行+隔离+双审查），强耦合链路走 Executing-Plans（连续上下文+checkpoint 验证）
