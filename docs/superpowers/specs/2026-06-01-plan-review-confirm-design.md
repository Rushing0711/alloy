# Plan 审查窗口 (b) 选项二次确认设计

## 问题

`/alloy:plan` 的制品审查窗口中，用户选 (b) "需要调整"后，AI 直接判断修改性质并执行（typo 直接改 / 需求变更回溯 brainstorming），没有给用户确认机会。

## 设计方案

### 流程变更

**现有流程：**

```
用户选 (b) → 说明修改点 → AI 判断 typo/需求变更 → 直接执行
```

**新流程：**

```
用户选 (b) → 说明修改点 → AI 内部判断 → 呈现确认选项 → 用户选择 → 执行
```

### 审查窗口新格式

审查窗口的 (a)/(b) 初始选项不变。用户选 (b) 并说明修改点后，AI 内部评估修改性质，然后呈现：

```
→ (a) 确认变更，回溯到 brainstorming
→ (b) 取消变更，继续当前审查
```

AI 判断只是内部推理过程，不对外展示标签（"这是 typo/需求变更"）。用户始终在 (a) 回溯和 (b) 取消之间选择。

### 适用范围

Plan 阶段所有制品审查窗口：proposal / design / specs / tasks / plans。

### AI 内部判断标准

- **typo/措辞修正**（可直接修改）：错别字、格式调整、表达优化，不改变功能边界
- **需求层面变更**（需回溯 brainstorming）：功能增删、行为变更、范围调整

无论 AI 如何判断，用户始终看到相同的 (a)/(b) 两个选项。

### 不改变的部分

- 初始审查窗口的 (a) 确认 / (b) 需要调整 选项保持不变
- 回溯清理步骤保持不变
- hash 锁定和 commit 流程保持不变
